# GLYPH — Progressive Discovery

**Status:** PRODUCT PHILOSOPHY (Executive Producer) — **supersedes onboarding-as-primary-frame**  
**Audience:** Design, production, CEO, playtest facilitators  
**Implementation:** None authorized by this document

**This is a learning philosophy, not a tutorial philosophy.**

> Players should always feel:  
> **"I understand this."**  
> Immediately followed by:  
> **"Wait… there's more."**

**Do not think in levels. Think in layers of player understanding.**

**Goal:** A game that **grows with the player's understanding** — never overwhelming, never finished learning.

**Production rule (unchanged):** *Can the player discover this instead?* → Discovery wins.

**Metric (refined):** Sustained curiosity — not immediate mastery, not strategic clarity at twenty minutes.

---

## What Progressive Discovery is

| Tutorial philosophy | Progressive Discovery |
|---|---|
| Front-load explanation | Front-load **competence at one layer** |
| Teach all rules early | Reveal rules **when the prior layer is lived** |
| Player is student | Player is **director learning their craft** |
| Completion = done learning | Completion = **door opens** |
| Onboarding is a phase | Onboarding is **the first layers of a lifelong curve** |

**Progressive Discovery** is how GLYPH exposes depth **in pace with lived understanding** — through board, hand, aftermath, and honest surprise — not through menus, levels, or lectures.

**Relationship to prior work:**

| Prior frame | New status |
|---|---|
| The First Five Performances | **First five layers** — not a separate "onboarding mode" |
| Aspiration analysis | **One emotion** inside Progressive Discovery — not urgent fix |
| Strategic uncertainty analysis | **Healthy** until curiosity collapses |
| Teach mastery without instruction | **Method** — mirror not teacher, at each layer |

## The rhythm (every layer)

Each layer follows the same beat:

```
COMPETENCE  →  CONFIDENCE  →  EDGE CASE  →  "WAIT, THERE'S MORE"  →  NEW LAYER
     ↑                                                              |
     └──────────────────── next layer feels like extension ─────────┘
```

- Player never feels **overwhelmed** (one new dimension at a time)
- Player never feels **finished** (ceiling always visible ahead)
- New layer **recontextualizes** old layer — never invalidates it

**CEO Performance 1 example:**

- *"I understand this"* — connection, lit/cold, 7/8  
- *"Wait… there's more"* — strategy feels opaque, how do I improve?  
- **That second feeling is not a bug.** It is Layer 1 closing and Layer 2–3 knocking — **if curiosity persists** (CEO: *"trying different things"* ✅)

## Question 1 — What are the natural layers of understanding?

Layers are **shifts in what the player perceives** — not XP tiers.

### Tier A — Single board (minutes → first hours)

| Layer | Unit of perception | Player thinks | Natural "wait, there's more" |
|---|---|---|---|
| **A0 — Agency** | Hand + cell | *"I place. I spark."* | *"What happens when I spark?"* |
| **A1 — Connection** | Touching vs isolated | *"Only connected runes fire."* | *"Does WHERE I touch matter?"* |
| **A2 — Directorship** | Spark origin | *"Same board, different show."* | *"Why did THAT one behave differently?"* (elements) |
| **A3 — Element character** | F / L / W / C behavior | *"They don't spread the same way."* | *"What happens when they meet?"* |
| **A4 — Conversion** | Steam, combos | *"Water + fire did something extra."* | *"Why was the END worth so much?"* |
| **A5 — Multiplier curve** | Late-chain weight | *"The tail of the chain matters most."* | *"Can it go MORE than one direction?"* |
| **A6 — Branching** | Forks, crystal fans | *"The wave splits."* | *"When do I stop building?"* |
| **A7 — Stop discipline** | When to Spark | *"Too early / too crowded."* | *"Can I make ANY hand work?"* |

### Tier B — Session (hours)

| Layer | Unit of perception | Player thinks | "Wait, there's more" |
|---|---|---|---|
| **B1 — Hand reading** | Offered tools as puzzle | *"This hand wants a shape."* | *"How do I judge without full sim?"* |
| **B2 — Heuristics** | Fast board judgment | *"This layout feels better."* | *"I almost had a principle."* |
| **B3 — Motif library** | Reusable sub-shapes | *"I've seen this before."* | *"Same motif, new constraint?"* |

### Tier C — Mastery (tens → hundreds of hours)

| Layer | Unit of perception | Player thinks | "Wait, there's more" |
|---|---|---|---|
| **C1 — System view** | Board as graph | *"I see routes, not runes."* | *"There's a better route."* |
| **C2 — Marginal optimization** | Extraction from identical hand | *"Same tools, more points."* | *"What would a master do?"* |
| **C3 — Principles** | Rules behind motifs | *"Principles, not recipes."* | Phase 7 constraints, new topology |
| **C4 — Adaptation under constraint** | New rules, old mind | *"Old habits almost work."* | New constraint families |

### Tier D — Product (post–vertical slice)

| Layer | Source | Never in slice |
|---|---|---|
| **D1+** | Objective types, topology, run structure, meta breadth | Phase 6–7 full game |

**Frozen mechanics support Layers A0–A7 in the slice.** Phase 3 validates B–C structurally. Phase 7 supplies D.

## Question 2 — In what order should they emerge?

**Rule:** Each layer must be **lived** before the next is **invited**. Order follows **dependency of perception**, not designer convenience.

### Canonical emergence sequence

```
A0 Agency
  ↓
A1 Connection          ← CEO Performance 1 / P1
  ↓
A2 Directorship (spark origin)
  ↓
A3 Element character   (can partially co-emerge with A2 via contrast)
  ↓
A4 Conversion
  ↓
A5 Multiplier curve
  ↓
A6 Branching
  ↓
A7 Stop discipline
  ↓
B1 Hand reading
  ↓
B2 Heuristics → B3 Motifs → C1 System → C2 Optimization → C3 Principles → C4 Constraints
```

### Order principles

1. **Local before global** — connection before multiplier math  
2. **See before optimize** — element behavior before extraction theory  
3. **One board before session** — A layers before B layers  
4. **Honest surprise after predictability** — conversion/branching after basic propagation  
5. **Never skip A0–A1** — agency and connection are foundation  

### First Five Performances → first layers (reframed)

Performances are **not** a tutorial track. They are **the first authored invitations into Layer A**:

| Performance | Layer invitation (not lesson) | Question created |
|---|---|---|
| P1 | A1 Connection | *"Will the lonely one fire?"* |
| P2 | A2 Directorship (hypothesis) | *"Does it matter where I start?"* |
| P3 | A3 + reach (hypothesis) | *"Can it cross that gap?"* |
| P4 | A6 Branching (hypothesis) | *"Can it split?"* |
| P5 | Vision — combinatorial self-surprise | *"I didn't know I could do that"* |

**Performances 2–5 remain hypotheses** — authored from evidence, not locked (EP prior directive).

**After Performance 5:** Progressive Discovery continues via **procedural hands + mirror** — not more performances forever.

## Question 3 — How do we know a player has mastered one layer before exposing the next?

**Not:** quizzes, level gates, "tutorial complete" flags.  
**Yes:** **observable understanding behaviors** — same evidence discipline as Constitution §13.

### Layer mastery signals (facilitator / playtest)

| Layer | Mastery looks like | Not ready yet |
|---|---|---|
| **A0** | Places and Sparks without help | Confusion about loop |
| **A1** | Explains lit/cold via connection unprompted | *"Am I playing right?"* |
| **A2** | Varies spark origin intentionally | Never changes origin |
| **A3** | Names element **behavior** in own words | Treats all runes as same |
| **A4** | References steam/conversion unprompted | Never sees conversion |
| **A5** | Says *"the end mattered more"* | Only looks at total count |
| **A6** | Builds toward fork intentionally | Surprised by every branch |
| **A7** | Spark timing varies with board state | Always fires immediately |

### Product exposure (how the game reveals — not when UI unlocks)

| Mechanism | Role |
|---|---|
| **Curated performances / hands** | Invite **one** new layer when evidence says prior layer lived |
| **Element in hand** | Introduce A3+ when connection + origin stable |
| **Board mirror** | Confirms layer N so layer N+1 can be **felt** |
| **Constraint variety (full game)** | Reopens C4 without new tutorial |
| **Never:** hard locks that say "complete lesson 2" | Violates discovery philosophy |

### CEO gate (current)

**Layer A1:** ✅ Mastered (connection, experimentation)  
**Layer A2–A7:** Not yet invited in product — **correct**  
**"Wait, there's more":** ✅ Present (strategy opacity)  
**Curiosity:** ✅ Present (*trying different things*)

**Do not expose A2–A7 because CEO said "not sure what strategy is."** That is **Layer A1 closing**, not a signal to dump strategy.

## Question 4 — Can every layer create a new "aha" moment?

**Yes — if each layer is designed as extension, not appendix.**

| Layer | "Aha" shape | Example unprompted quote |
|---|---|---|
| A1 | **Exclusion** | *"The lonely one didn't fire."* |
| A2 | **Same → different** | *"I should've sparked from the other end."* |
| A3 | **Identity** | *"Lightning jumped the gap!"* |
| A4 | **Synergy** | *"The water exploded when fire hit it."* |
| A5 | **Economics** | *"The last ones were worth way more."* |
| A6 | **Topology** | *"It went BOTH ways."* |
| A7 | **Timing** | *"I should've fired earlier."* |
| B+ | **Pattern** | *"I've seen this shape before."* |
| C+ | **Principle** | *"Same hand — I left points on the board."* |

### Aha requirements (Constitution-safe)

- **Honest** — §1 never fake the aha  
- **Hindsight-explainable** — board shows why after  
- **Player-owned** — they authored the board  
- **One dimension** — not five revelations at once  

### When aha fails

- Layer skipped (branching before connection)  
- Layer told (popup explains steam before player sees steam)  
- Layer fake (scripted near-miss)  
- Layer drowned (too many elements in hand too early)

## Question 5 — Can this progression continue for dozens of hours?

**Yes — structurally.** Pre-production validated this explicitly.

| Source | What it proves |
|---|---|
| **Phase 3** | 1h → 10h → 100h → 1000h perception shift (rune → combo → system → principle) |
| **Phase 2** | Combinatorial depth past mental simulation — 70/80 predict, 20/30 surprise |
| **Phase 7** | Constraint variety reopens space without new mechanics |
| **Determinism** | Skill compounds — not solved in one afternoon |
| **Variable hands** | Cannot memorize — must understand |

### Hours map (illustrative — not levels)

| Hours | Dominant layers | Player feeling |
|---|---|---|
| **0–1** | A0–A2 | *"I understand this… wait, there's more."* |
| **1–5** | A3–A7 | Element depth, branching, stop discipline |
| **5–20** | B1–B3 | Hand reading, motifs — strategic fog **healthy** |
| **20–100** | C1–C2 | System view, optimization |
| **100+** | C3–C4 + D | Principles, constraints, meta breadth |

**CEO at ~20 minutes:** Layer A1 competent, A2+ uninvited — **on schedule** for Progressive Discovery, **behind** for "immediate strategy" (which is not the goal).

### Why player never feels finished

- Multiplier + branching + hand variance → no perfect board  
- Constraint variety (full game) → old principles in new clothes  
- Personal best + target → self-referential ceiling always rises  
- Phase 3: same board, Theorist vs Reactor — **same rules, unbounded extraction**

### Why player never feels overwhelmed

- **One layer per beat** — performances/hands/encounters invite one new question  
- **Calm pacing** — Creative Bible; no FOMO timers  
- **Mirror confirms prior layer** before next invites  
- **Strategic fog at A1** does not require **C1** knowledge

## Product implications (philosophy — not build list)

### Do

- Design every feature as **Layer N invitation** or **Layer N mirror**  
- Measure **which layer** player is living (playtest quotes)  
- Let **evidence** author next performance / hand bias — not designer impatience  
- Preserve *"I understand this / wait, there's more"* rhythm every session  
- Keep Performances 2–5 as **hypotheses** until prior layer mastery observed  

### Do not

- Reveal strategy immediately  
- Collapse uncertainty + curiosity into "problem to fix"  
- Rush aspiration systems to solve Layer B fog at Layer A  
- Use levels, XP, or "lesson complete" UI  
- Expose all four elements + multiplier + branching in Performance 1  
- Treat onboarding as finished after Performance 5 — **Progressive Discovery never ends**  

### Vertical Slice scope (unchanged)

Slice proves **Hope Contract + first layers of Progressive Discovery** (A0–A1 minimum, ideally through A4–A6 in first session if evidence invites). **Not** full C-layer mastery.

| *"other end / where I started / which one I tapped"* | A2 |

| *"end mattered / multiplier / bigger at the tail"* | A5 |
