# GLYPH — Milestone Review Protocol

**Authority:** CEO production rule (2026-06-29)  
**Applies to:** Every production milestone — no exceptions

---

## Rule

Every production milestone **ends with a playable build**.

**Production does not continue** into the next milestone until the CEO has:

1. **Played** the latest build
2. **Reviewed** what changed
3. **Answered** targeted review questions

---

## Every milestone deliverable must include

| Deliverable | Required content |
|---|---|
| **Playable build** | Tagged version string · link or run instructions |
| **Build notes** | What changed · what did not change · constitutional alignment |
| **Known issues** | Honest list — no hiding blockers |
| **Review questions** | Specific, testable questions for CEO playtest |

---

## Question discipline

**Do not ask:** *"What do you think?"*

**Do ask:** Targeted questions tied to evidence gates and VS-FEELING minutes.

Examples:

- Did the frontier feel more alive?
- Did the Hope Pass improve?
- Did the first five minutes feel better?
- Did you voluntarily replay?
- What frustrated you?

---

## Flow

```
Milestone complete → playable build + build notes + questions
        ↓
CEO plays → answers questions
        ↓
Milestone review together → authorize or revise
        ↓
Next milestone (one sprint only)
```

**Goal:** Hundreds of small, evidence-based improvements — not massive redesigns.

**The CEO plays every milestone before production continues.**
