# GLYPH — The First Five Performances

**Internal name:** The First Five Performances *(formerly Discovery Sequence)*  
**Status:** **APPROVED** (Executive Producer) — **no implementation authorized yet**  
**Audience:** Design, production, playtest facilitators

**Do not think like a teacher. Think like a puzzle designer.**

**Production rule:**

> Before proposing any explanation, ask: *"Can the player discover this instead?"*  
> **If yes — discovery wins.**

**Constitution anchor:** Agency · trust · hope · player intelligence.

**CEO north star:** *"I intentionally left runes disconnected to see what would happen."* — protect this forever.

---

## What this is

The First Five Performances are **not lessons**. They are the player's **first five authored performances** — handcrafted openings where each board poses a **question**, the player answers through **experiment**, and the game **almost never answers directly**.

| A tutorial | The First Five Performances |
|---|---|
| Explains | Poses questions |
| Teaches a rule | Creates curiosity |
| Answers | Creates opportunities to discover |
| Completes | Graduates |

**Graduation is not tutorial completion.**

Graduation is Board 5 — when the player thinks:

> *"I didn't know I could do that."*

---

## Design philosophy

### Puzzle designer questions (not teacher lessons)

**Never ask internally:** *"What should Board 1 teach?"*

**Always ask:** *"What question should Board 1 make the player ask?"*

The player answers by experimenting. The board mirror (lit/cold, spark origin) confirms or refutes — it does not lecture.

### The game almost never answers directly

- No modals · no arrows · no rule popups · no "correct solution"
- ≤3 lines shell copy total before first Spark — **invitation**, not instruction
- Aftermath is a **mirror** (*what happened?*), not a teacher (*here's why*)

### Protect curiosity

Any feature proposal that **explains** must survive:

1. *Can the player discover this instead?*
2. If yes → **do not explain**
3. If no → find the **smallest mirror** (board state, not copy)

---

## The five performances

**Rules for all five:**

- Handcrafted **forever** (GLYPH's 1-1 — fixed, evidence-comparable)
- Sparkable in under 90 seconds
- Safe to fail — no eject, no shame copy
- Player never sees "Performance 3 of 5"
- One **dominant question** per board — no mechanic pile-ups

---

### Performance 1 — *"Will the lonely one fire?"*

**Question the board poses:** *If I leave one rune apart from the others… will it still fire?*

| Puzzle design | |
|---|---|
| **Hand** | Enough runes to form a **pair** and an **orphan** — player must choose whether to connect |
| **Board** | Empty slate — full authorship of layout |
| **Intended experiment** | CEO behavior by design: disconnect on purpose |
| **Mirror answers** | Lit pair vs cold orphan — no text |

**Player discovers (if they experiment):** Connection matters. Not because we said so — because the board showed it.

**Curiosity preserved if:** Disconnected placement is never punished or labeled wrong.

---

### Performance 2 — *"Does it matter where I start?"*

**Question the board poses:** *Same stones — does it matter which one I call Action on?*

| Puzzle design | |
|---|---|
| **Hand** | Minimal — extend or don't |
| **Board** | Pre-seeded **line of three** touching runes — asymmetric element order so ends differ |
| **Intended experiment** | Spark left end → replay or re-spark mental note → spark right end |
| **Mirror answers** | Different lit sets; spark origin mark persists |

**Player discovers:** Spark origin is a directorial choice — same stage, different performance.

**Not:** A lesson called "Spark Origin."

---

### Performance 3 — *"Can it reach across that gap?"*

**Question the board poses:** *There's empty space between these clusters — can the chain cross it?*

| Puzzle design | |
|---|---|
| **Hand** | Fire + Lightning + support — player chooses which tool to deploy |
| **Board** | Two clusters, **one empty cell** between — puzzle geometry invites the question |
| **Intended experiment** | Try Fire first (gap stops it) · try Lightning (arc crosses) |
| **Mirror answers** | One-gap cold vs arc travel line — factual, not labeled |

**Player discovers:** Some elements reach; some don't. Hindsight explainable (§1).

---

### Performance 4 — *"Can it go more than one way at once?"*

**Question the board poses:** *If I set this up right… will the wave split?*

| Puzzle design | |
|---|---|
| **Hand** | Crystal + kindling — enough to build a fork, not enough to force one solution |
| **Board** | Empty or single seed — player authors toward a Y/T |
| **Intended experiment** | Build → Spark → watch frontier branch |
| **Mirror answers** | Multiple lit arms; chain HUD climbs; hope text may appear |

**Player discovers:** Branching exists — hope enters as participatory rooting (Hope Contract).

**This is the first performance where Minute 3 (hope) should reliably ignite.**

---

### Performance 5 — *"I didn't know I could do that."* (Graduation)

**Question the board poses:** *No new question. Open stage.*

**This performance does NOT combine previous mechanics like a final exam.**

It is the first board where **combinatorial depth exceeds the player's mental model** — and the outcome is **fully honest** (deterministic, traceable on the board after).

| Puzzle design | |
|---|---|
| **Hand** | Full four-element palette — first time all four available |
| **Board** | Empty — pure authorship |
| **Objective** | Target visible — player has context from four prior performances |
| **Design target** | Local plan ("I planned that") + global surprise ("I didn't compute *that*") — Phase 2's 70/80 · 20/30 split |
| **Graduation moment** | Unprompted *"I didn't know I could do that"* — not *"Tutorial complete"* |

**Player discovers:** They are capable of authoring something beyond their expectation. **That is graduation.**

**Failure is OK:** Mirror still answers *what happened*; voluntary replay still available. Graduation is emotional, not "beat target."

---

## Performance arc (internal — never player-facing)

```
P1: "Will the lonely one fire?"
P2: "Does it matter where I start?"
P3: "Can it reach across that gap?"
P4: "Can it go more than one way?"
P5: "I didn't know I could do that."  ← graduation
```

Player feels: five chances to **direct**, not five lessons to **pass**.

---

## What we rejected (unchanged)

- Traditional tutorial · playable tutorial · rule encyclopedia
- Board 5 as "combine all lessons" exam
- Explaining when discovery is possible
- Labeling disconnected runes as mistakes

---

## Success behaviors (curiosity, not comprehension)

| Signal | Meaning |
|---|---|
| Voluntary disconnected placement | Curiosity protected (CEO behavior) |
| Voluntary second Spark | Loop landed |
| Player changes spark origin between attempts | P2 question self-answered |
| *"I didn't know I could do that"* on P5 | **Graduation** |
| Absence of *"Am I playing right?"* | Mirror working |
| Hope language during P4–P5 | Hope Contract alive |

**Success is NOT:** player recites four element rules.

**Success IS:** player keeps experimenting because the **questions** are more interesting than the answers.

---

## After Performance 5

| Scope | Approach |
|---|---|
| Performances 1–5 | Handcrafted forever — once per player |
| Board 6+ | Procedural hands — variety, mastery, replay |
| Full product | Phase 7 constraint variety — post-slice |

The opening five are the **opening movement**. The symphony is procedural.

---

## Dependencies (production)

Before implementing Performances:

1. **Board mirror legibility** — lit vs cold must be obvious; P0 taxonomy may need revision if too subtle
2. **Performance data format** — handcrafted hand + optional seed board as config (implementation later)
3. **EP/CEO lock** on Performance 5 puzzle spec — must be authored to produce self-surprise, not tested once

---

## Decision checklist (every proposal)

Before any copy, UI, or "help" feature ships:

- [ ] What **question** does this create?
- [ ] Can the player **discover** the answer instead?
- [ ] If we explain, is there **no** mirror alternative?
- [ ] Does this protect or kill **voluntary experimentation**?
- [ ] Which constitutional principle does this strengthen?

**If discovery is possible — discovery wins.**
