# GLYPH — Playtest Protocol v1

**Experiment:** Hope Pass (`experiments/glyph-spark-test/hope-pass.html`)  
**Build status:** FROZEN — do not modify the build during this protocol.  
**Purpose:** Evidence collection only. Not design. Not implementation.

**Emotional contract under test:**

> The player directs a performance they authored. They hope the frontier reaches one more rune. The game never lies about whether it did. Pride lands before the numbers.

---

## Before you begin

### Facilitator rules

- **Say only this to start:** *"Place runes on the board, then click one to spark. Try to connect as many as you can. I'll ask a few questions when you're done."*
- **Do not** explain rune rules unless the tester asks directly — then answer briefly and truthfully.
- **Do not** coach, hint, or suggest strategies during play.
- **Do not** ask *"Was it fun?"* at any point.
- **Observe silently** unless the tester asks a factual question about controls.
- **Record exact quotes** when possible — in quotation marks, verbatim.
- **One tester per session.** No group play for Protocol v1.

### Tester setup

- Open `hope-pass.html` in a browser (full screen if possible).
- Sound **on** unless tester prefers off — note preference on sheet.
- Facilitator has a copy of this sheet (print or second screen).
- Session timer ready.

## Success / failure signals (for aggregation)

**Protocol v1 — session counts as HOPE PASS EVIDENCE POSITIVE if:**

- [ ] Tester used hope language unprompted (Q3 yes, or quote log)
- [ ] ≥2 voluntary replays in 10 minutes
- [ ] Post Q2 references watching chain/frontier, not only numbers
- [ ] Post Q4 cites a reason beyond "higher score" (e.g. connect more, fix layout, see if it reaches)

**Protocol v1 — session counts as HOPE PASS EVIDENCE NEGATIVE if:**

- [ ] One run and done with no voluntary replay
- [ ] Post Q2/Q3 focused only on score/target
- [ ] Post Q5 cites boredom or confusion with no recovery
- [ ] No experiment within 10 minutes
